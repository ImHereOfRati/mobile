# 사용자 노출 용어 매핑표 & 카피 가이드

ui-transition-plan Phase 1 결과물.
원칙: 내부 구현명(geofence 등)은 코드에 유지하고, 사용자 노출 텍스트만 교체한다.

| geofence 등록 (enroll) | 알림 만들기 | 첫 생성 CTA는 `첫 알림 만들기` |
| eventType: arrival | 도착했을 때 | |

## 화면별 카피 기준

### 로그인 (auth)
- 서비스 가치 중심: `위치 기반 자동 연락 서비스` 인상.
- 권한 나열형 설명 금지.

### 약관 (terms)
- 필수 동의 중심: `필수 약관에만 동의하면 바로 시작할 수 있어요`.
- 보조 문구: `선택 항목은 동의하지 않아도 이용 가능해요`.

### 메인 (geofence list)
- 목적: 내 알림 관리. 헤더 `내 도착 알림`.
- 구조: 페이지 설명 → 준비 상태 카드 → 알림 리스트 → `새 알림 만들기`.
- empty state: `아직 만든 도착 알림이 없어요` + `첫 알림 만들기` CTA.
- 카드 상태 라벨: `자동 전송 준비 완료` / `준비 필요` / `자동 전송 꺼짐`.

### 등록 (enroll)
- 질문형 섹션 순서: 어떤 친구에게 알려줄까요? → 언제 알려드릴까요? → 어디에서 알려드릴까요? → 반복 설정 → 장소 이름 → 보낼 메시지 (선택) → 저장.
- 지도 힌트: `알려드릴 장소를 지도에서 눌러 선택하세요`.

### 등록 완료 (complete sheet)
- 제목: `{도착|출발|도착/출발} 알림이 저장되었어요!`
- 요약: `{장소}에 {도착하면} {수신자}(외 N명)에게 자동으로 알려드릴게요.`
- CTA: 자동 전송 켜기(미준비 시) / 알림 더 만들기 / 메인으로 돌아가기.

### 설정 (setting)
- 목적: 문제 해결. 최상단 자동 전송 준비 카드.
- 상태 라벨: 준비 완료 / 설정 필요 / 설정에서 변경 필요 / 제한됨 / 상태 확인 필요.
- 전송 진단 섹션: 마지막 자동 전송(로컬 기록) + 제조사 절전 주의 + 진단 placeholder.

### 권한 안내
- 위치 항상 허용: `자동 전송을 사용하려면 위치를 항상 허용으로 바꿔야 해요.`
- 배터리 최적화 제외: `앱이 닫혀 있어도 자동 전송이 끊기지 않게 배터리 최적화 제외가 필요해요.`

## 금지 용어 (사용자 노출)
