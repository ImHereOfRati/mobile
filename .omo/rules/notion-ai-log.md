# ImHere Notion AI Log Rule

이 프로젝트에서 작업을 마치면, Notion MCP 가 사용 가능할 경우 반드시 `ImHere 프로젝트` 아래의 `AI 기록` 공간에 작업 기록을 남긴다.

## 1. 기록 위치 규칙

1. 최상위 기준은 `ImHere 프로젝트` 페이지다.
2. 그 아래에 `AI 기록` 페이지 또는 데이터베이스가 있으면 그 안에 기록한다.
3. `AI 기록`이 없으면 먼저 만든다.
4. 기록은 하루에 하나의 페이지를 사용한다.
5. 일별 페이지 제목은 `YYYY-MM-DD` 형식을 기본으로 한다.

## 2. 일별 페이지 규칙

1. 같은 날짜 페이지가 있으면 새로 만들지 말고 그 페이지에 이어서 기록한다.
2. 같은 날짜 페이지가 없으면 새 페이지를 만든다.
3. 일별 페이지 상단에는 날짜와 프로젝트 이름만 간단히 유지한다.

## 3. 작업 단위 기록 규칙

1. 한 번의 사용자 요청 또는 한 묶음의 연속 작업을 하나의 기록 단위로 남긴다.
2. 일별 페이지 안에서는 작업마다 별도 섹션 또는 토글 블록을 만든다.
3. 섹션 제목은 `HH:mm - 작업 요약` 형식을 기본으로 한다.

## 4. 각 작업 기록 필수 항목

각 작업 기록에는 아래 항목을 반드시 남긴다.

1. `Prompt`
: 사용자가 실제로 요청한 핵심 프롬프트 또는 요약
2. `Model`
: 실제 사용 모델명
3. `Problem`
: 해결하려던 문제, 목표, 제약
4. `Procedure`
: 어떤 절차로 해결했는지 단계별 요약
5. `Issues`
: 작업 중 마주친 문제, 실패, 막힌 점
6. `Resolution`
: 문제를 어떻게 해결했는지
7. `Prompt Evaluation`
: 프롬프트가 명확했는지, 모호했는지, 다음에는 어떻게 더 좋아질지

## 5. 기록 품질 규칙

1. 기록은 짧더라도 재현 가능하게 쓴다.
2. 단순 성공만 적지 말고, 중간 실패와 수정 포인트도 남긴다.
3. 민감정보, 토큰, 비밀번호, 개인식별정보는 기록하지 않는다.
4. 사용자의 원문 요청이 너무 길면 핵심만 축약하되 의미를 바꾸지 않는다.

## 6. MCP 사용 규칙

1. Notion MCP 가 연결되어 있으면 작업 종료 전에 기록을 남긴다.
2. Notion MCP 인증이 풀렸거나 접근 권한이 없으면 임의로 성공 처리하지 않는다.
3. 기록에 실패하면 최종 응답에서 실패 사실과 원인을 분명히 알린다.
4. 같은 작업에 대해 중복 페이지를 만들지 않는다.

## 7. 기본 기록 템플릿

```md
## HH:mm - 작업 요약

- Prompt: ...
- Model: ...
- Problem: ...
- Procedure:
  1. ...
  2. ...
  3. ...
- Issues: ...
- Resolution: ...
- Prompt Evaluation: ...
```
